## Database Schema (PostgreSQL via Drizzle)

### Tables

| Table | Purpose |
|-------|---------|
| `players` | User profiles: phone (unique), name, positions (jsonb), dominantFoot, selfSkills (jsonb), birthDate, alias, shirtNumber, photoUrl |
| `sessions` | Auth sessions: playerId (FK), expiresAt (30 days) |
| `player_ratings` | Crowdsourced ratings: raterId, ratedId, skills (jsonb). Unique on (raterId, ratedId) |
| `groups` | Clubs: name, logoUrl, city, country, description, createdBy |
| `group_members` | Club membership: groupId, playerId, role (admin/player). Unique on (groupId, playerId) |
| `matches` | Match instances: groupId, date, location, locationUrl, format, maxPlayers, enrollmentDeadline, status, createdBy |
| `match_enrollments` | Enrollment: matchId, playerId, status (enrolled/waitlisted/removed), joinedAt |
| `lineups` | AI lineups: matchId, teamA (jsonb), teamB (jsonb), bench (jsonb), justification, published |
| `match_results` | Scores: matchId, scoreA, scoreB |
| `match_events` | Events: matchId, playerId, type (goal/assist/injury/mvp_vote), meta (jsonb) |

### Enums

- `dominant_foot`: left, right, both
- `group_role`: admin, player
- `match_format`: 5v5, 7v7, 8v8, 11v11
- `match_status`: open, closed, in_progress, completed
- `enrollment_status`: enrolled, waitlisted, removed
- `match_event_type`: goal, assist, injury, mvp_vote

### Migrations

```bash
npx drizzle-kit generate   # Generate migration from schema changes
npx drizzle-kit push        # Push schema directly (dev)
npx drizzle-kit migrate     # Run migrations (prod)
```

## API Routes

### Auth
- `POST /api/auth` — Login: find-or-create player by phone, create session
- `DELETE /api/auth` — Logout: destroy session

### Players
- `GET /api/players` — List (paginated, searchable)
- `POST /api/players` — Create profile
- `GET /api/players/[id]` — Detail + ratings summary
- `PATCH /api/players/[id]` — Update profile
- `DELETE /api/players/[id]` — Delete
- `GET /api/players/me` — Current player

### Ratings
- `POST /api/ratings` — Upsert skill rating (one per rater/rated pair)

### Matches
- `GET /api/matches` — List with filters
- `POST /api/matches` — Create (admin)
- `GET /api/matches/[id]` — Detail + enrollments
- `PATCH /api/matches/[id]` — Update status/details

### Enrollments
- `POST /api/enrollments` — Join/leave/waitlist

### Lineups
- `POST /api/lineup/generate` — Claude API lineup generation (admin, max 5 per match)
- `GET /api/lineup/[matchId]` — Get published lineup

### Clubs
- `GET /api/groups` — List (filterable by city/country)
- `POST /api/groups` — Create club
- `GET /api/groups/[id]` — Detail
- `PATCH /api/groups/[id]` — Update (admin)
- `GET /api/groups/[id]/members` — List members
- `POST /api/groups/[id]/members` — Add/remove member

### Other
- `POST /api/admin/players` — Admin-only player creation
- `POST /api/uploads/club-logo` — Image upload

## Auth Flow

1. User enters phone number with country picker (MX +52, CL +56, AR +54)
2. `POST /api/auth` normalizes phone, finds or creates player, creates session (30-day expiry)
3. Session stored in `sessions` table, `session_id` set as httpOnly cookie
4. `middleware.ts` checks cookie on every request — redirects to `/login` if missing
5. `getSession()` in `src/lib/auth.ts` reads cookie, joins sessions+players, validates expiry

**Note:** No SMS/OTP verification. Phone number is trusted on input.

## Lineup Generation

- Prompt built in `src/lib/claude/lineup.ts`
- Two modes: "both teams" (balanced split) or "single team" (best XI from pool)
- Player locking: admin can lock specific players to a team before regeneration
- Max 5 regenerations per match
- Model: `claude-sonnet-4-6`
- Output: structured JSON parsed with Zod, stored in `lineups` table
- Visual display: SVG pitch formation in `lineup-view.tsx`
